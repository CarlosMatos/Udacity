Lesson 1 - HTML definition

HTML - What animal is it?

HTML is the basis for most of the WEB pages. It is a set rules to put together different elements, like text, images, videos and links, to be showed in the PC screen throw a WEB browser. Or, in other words, HTML is a program code that a WEB browser is able to interpret in order to show some content.

A markup language.

As every program code, HTML supposes a syntax to be followed by the programmer. Basically, HTML uses some special signs to instruct the browser what type of effect it should apply to the content. Those signs are generically called tags and they have the format <EFFECT>. A tag, in general (there are exceptions) are used in pairs: an opening tag, <EFFECT>, placed at the beginning of the content telling to the browser how the content shall be presented, and a closing tag </EFFECT> signaling to the browser were the effect applied finished. For instance, if one wants to write a piece of text in bold style, it should initiate the text with a special tag <b> and finalize it with </b>. The text in between will be emphasized in bold style.

Elements

An Element in HTML is piece of code composed of a pair of tags and the content in between. The tags used in an Element will differentiate the way it is showed in the screen.

Attributes

The opening tag can contain other items, called attributes. When the type of effect applied need more complements, those attributes will provide then. For instance, if we want to instruct that some text is a link to other page, we use the tag <a>, plus attributes telling the browser the direction it should appoint if the user click on the link. So

<a href= www.outrapagina.com.br> here goes the text that will be seen as a hyperlink</a>

will transform the text here goes the text that will be seen as a hyperlink in a piece of text that one can click on it to accessthe URL www.outrapagina.com.br .

Lesson 2: Using HTML to generate documents in a structured format

Does the HTML need to be structured to work?

In fact it doesn't, but to produce programs in a structured format helps the programmer and others to understand better and faster the logic built in.

The steps to structure a HTML program.

Since HTML are "boxes inside boxes", the first step is to identify on the document draft or scratch how many boxes one will need to build a HTML document and what is the hierarchy among them, or in other words, which box contains other boxes, being the box that enclose others having more hierarchy than those enclosed.

In the HTML syntax, some elements, like h, p, div, are used to delimit boxes of content. Those elements shall be used to implement the content boxes identified during the first step. And to highlight the hierarchy of the boxes, is a recommended practice to indent those elements, with the level of indentation corresponding to the level of hierarchy the box has in the overall structure. An example is the way how this text is formatted.

We can organize this text in boxes as:

  1. one box for the whole document,
  2. one box for each lesson title,
  3. one box for each text paragraph title,
  4. one box for text paragraph and
  5. one box for this list

We can hierarchize these boxes in a decreasing order as: 1 with higher hierarchy, 2 lower, 3 lower and 4 and 5 at the same hierarchy being the lowest of them.

Lesson 3: Adding style to the HTML page

Do not repeat, do not repeat, do not repeat, and, once more, do not repeat!

One important point during the coding process is to avoid repeating code. Some reasons to avoid it are: it enlarges the code size unnecessarily, if you need to make some adjustment in the portion of the code that is repeated there are great possibility you forget to adjust all of them, it becames the code more difficulty to understand.

CSS

The style of the HTML sheet can be defined inside de HTML itself, but it isn't a good practice. The more usual and accepted practice is to write the effects to be applied in the HTML page in a CSS (Cascading Style Sheets) file. The term cascading means that the more specific style rule defined for an element in one of the style sheets (there may be several of them) will be applied for all same elements in whole HTML page.

This file must be mentioned in the <head> element, in HTML file, positioned before the <body>, using the tag <link>.

CSS uses selectors to identify in what elements of the HTML is applied a specific style.
